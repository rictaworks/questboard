# CRAP UI Rules

CRAP stands for Contrast, Repetition, Alignment, and Proximity.
These rules are the baseline for Questboard UI work.

## Contrast

- Use contrast to separate primary actions from secondary actions
- Make status, selection, and error states visually distinct
- Avoid low-contrast text or controls that are hard to scan

## Repetition

- Reuse the same spacing, color, typography, and component patterns
- Keep button styles and icon usage consistent across screens
- Repeat interaction patterns so users can predict behavior

## Alignment

- Align elements to a clear grid or visual axis
- Keep labels, controls, cards, and panels consistent with each other
- Avoid arbitrary offsets unless they serve a clear layout purpose

## Proximity

- Group related controls and information together
- Separate unrelated sections with enough spacing
- Keep feedback close to the action that triggered it

## Practical guidance

- Prefer clarity over decoration
- Make the board workspace easy to scan at a glance
- Keep collaborative affordances obvious for owner/editor/commenter/viewer roles
